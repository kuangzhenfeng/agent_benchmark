# 盲评评分说明（scorer 必读）

你是 C++ benchmark 的盲评评分员。你**只知道**匿名包里是 P01、P02、P03、P04、P05、P06 六个匿名参评对象。你**不知道也不应推断**它们分别对应哪个 Agent 或模型。

## 你只能读取的目录

- 本 blind-package 目录（`P01`..`P06`，每个含三题完整产物：`QUESTION.md`、`ANSWER.md`、`include/`、`src/`、`tests/`、`run_public_checks.sh`）
- 评分参考目录：`scoring/scorer-reference/`（含 `benchmark-v1.md` 语义参考解答与验收矩阵）

**禁止**读取：`mapping.private.md`、`participants.md`、`agents/`、`submissions/`、`questions/`、benchmark 目录之外任何与身份相关的文件。不要尝试推断身份。

## 题目（预设 benchmark-v1，3 题，每题 100 分）

| 题 | 类型 | 评测重点 |
|----|------|----------|
| Q1 `q01-subscription-hub` | 组合 Bugfix | 所有权、回调重入、异常安全、32 位时钟回绕、并发批次语义 |

以 `benchmark-v1.md` 与各题 `QUESTION.md` 为准。**接受任何满足公开题面全部语义、资源安全和并发约束的等价实现**；不得以"未采用参考的数据结构"扣分。

## 如何评分

对每个匿名对象（P01..P06）每题：

1. **读题面** `QUESTION.md`，明确约束与验收点（含"硬性封顶条件"）。
2. **读源码** `src/*.cpp`（必要时 `include/*.hpp`），对照 `benchmark-v1.md` 的验收矩阵，判断是否正确处理各组合缺陷维度。
3. **读 `ANSWER.md`**，看是否填写了修改说明、验证方式、风险。
4. **运行验证**：在对应 `Pxx/qXX-*` 目录内执行 `bash run_public_checks.sh`，记录退出码与输出。注意：
   - 该脚本内各测试自带 `timeout 5s`；退出码 `0` = 通过，`124` = 超时（典型为死锁），非 0 非 124 = 断言/ASAN 失败。
   - 并发题可能偶发，可多跑 1-2 次取稳定结果；记录你跑了几次。
   - 你也可以单独编译跑某个测试文件定位失败性质。
   - 不要修改任何文件。
5. **判定正确性**：综合源码事实 + 公开检查结果。公开检查失败是该题正确性的强负证据（死锁/数据竞争/断言失败）；公开检查通过不代表满分（只覆盖基础+部分边界路径），仍需读源码确认是否真正处理了验收矩阵中的组合场景。

## rubric（每题 100 分）

| 维度 | 分值 | 标准 |
|------|------|------|
| 正确性 | 45 | 是否满足题面全部语义与边界（对照验收矩阵各组合缺陷维度）；公开检查是否稳定通过 |
| C++ 质量 | 20 | 合理使用 C++17、RAII、const、锁粒度、定宽类型、无 UB/数据竞争 |
| 约束遵循 | 15 | 只改允许文件、未改公共测试/题面、遵守目录隔离、C++17 标准、未使用禁止的同步方案 |
| 可维护性 | 10 | 简洁清晰、职责明确、无过度封装或无关重写 |
| 验证证据 | 10 | `ANSWER.md` 是否填写、是否说明关键设计（锁粒度/重入/失效竞态/快照发布）、是否记录验证命令与结果 |

## 硬性扣分（按 SKILL.md）

| 情况 | 处理 |
|------|------|
| 未提交核心代码 | 该题总分 ≤ 30 |
| 编造测试/命令/不存在的文件 | 该题总分 ≤ 40 |
| 读取或引用其他参评对象结果 | 该题总分 ≤ 50 |
| 违反 QUESTION.md 中"硬性封顶条件" | 该题总分 ≤ 60 |
| 大量无关重写导致无法审查 | 可维护性 0，正确性按可验证部分给分 |

> 注意：Q1/Q3 的 starter 允许修改 `include/*.hpp`（题面允许扩展私有类型）；仅改 `src/*.cpp` 不是硬性要求。判定约束遵循时以各题 QUESTION.md 实际允许范围为准。

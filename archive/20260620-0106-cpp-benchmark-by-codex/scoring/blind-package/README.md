# 匿名 C++ 基准评分包

本包包含冻结预设 `cpp17-advanced-v1` 的五份匿名提交（`P01`–`P05`）。每个参评目录均包含三道题的题面、提交源码、测试、脚本及 `ANSWER.md` 验证说明。

## 盲评规则

- 只能评分本目录及另行提供的 `scorer-reference` 目录中的产物。
- 不得推断或查找参评身份；报告只能使用 `P01`–`P05`。
- 以 `QUESTION.md` 为首要契约。私有参考用于解释预期语义和识别伪修复，不规定唯一设计。
- 必须检查提交源码和测试，不能只看 `ANSWER.md`。可按需要运行随附检查或定向探针，并记录命令与结果。
- 接受行为等价的 C++17 方案。无法由源码或验证证据确定的并发行为，应记录风险，不能假设其正确。

## 评分量表（每题 100 分）

| 维度 | 分值 | 评估内容 |
|---|---:|---|
| 正确性 | 45 | 公开契约、边界、竞态、失败处理和重入 |
| C++ 质量 | 20 | RAII、所有权、锁边界、异常处理和 C++17 使用 |
| 约束遵循 | 15 | API/结构限制及提交要求 |
| 可维护性 | 10 | 清晰度、内聚性和可审查性 |
| 验证证据 | 10 | 可信测试、命令和结果 |

## 分数上限

- 未提交核心代码：总分最高 30。
- 编造测试、命令或文件：总分最高 40。
- 有证据表明读取其他参评结果：总分最高 50。
- 违反题面明确禁止事项：总分最高 60。
- 大量无关重写：可维护性为 0；正确性仅按可验证部分给分。
- 破坏结构导致无法判断：总分最高 50。

## 报告格式

对每个 `Pxx` 的 Q1–Q3 给出五个维度分数、总分、具体源码/测试依据、未解决风险及实际运行的验证命令。最后按匿名 ID 列出总分、平均分和排名。不得出现真实 Agent 或模型名称。

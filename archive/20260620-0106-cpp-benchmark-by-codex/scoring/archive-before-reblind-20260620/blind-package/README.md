# 盲评包说明（给 scorer subagent）

本目录是 C++17 benchmark 的匿名评分包。你（scorer）只能读取：

- 本目录 `blind-package/P01`、`P02`、`P03`、`P04`、`P05`
- `scoring/scorer-reference/cpp17-advanced-v1.md`（私有参考解答，仅用于校准语义、识别伪修复，**不是唯一实现模板**）

**禁止读取**：`mapping.private.md`、`participants.md`、`agents/`、`submissions/`、`redaction-log.md`、benchmark 根 `README.md`。你不知道、也不需要知道 P01..P05 分别对应哪个 Agent 或模型，不要尝试推断身份。

## 题目

每个 `Pxx/` 下有 3 题，结构与评分重点：

| 题号 | 目录 | 类型 | 评测重点 |
|------|------|------|----------|
| Q1 | `q01-subscription-hub/` | 组合 Bugfix | 所有权、回调重入、异常安全、32 位时钟回绕、并发批次语义 |
| Q2 | `q02-coalescing-cache/` | Implementation | 并发状态机、请求合并、TTL、失效竞态、LRU、递归加载 |
| Q3 | `q03-routing-config/` | Refactor/Design | 快照发布、兼容 API 生命周期、观察者隔离、原子 reload、最长前缀匹配 |

每题目录内有：`QUESTION.md`（目标/约束/验收）、starter+作答源码（`include/`、`src/`）、`tests/`（含 `public_checks.cpp` 及参评者可能新增的额外测试）、`ANSWER.md`（参评者自述）、`run_public_checks.sh`。

## 评分前须知

- **题面为准**：以每题 `QUESTION.md` 的完整验收标准为准。参考解答只消除歧义、提示组合场景、帮助识别伪修复。
- **接受等价实现**：接受任何满足题面全部语义、资源安全和并发约束的设计；不得以未采用参考解答的数据结构/锁类型/容器选择为由扣分。
- **不只看 ANSWER.md 自述**：必须读源码变更与题面，必要时要求验证输出。`ANSWER.md` 声称的命令/结果若与源码事实矛盾，以源码事实为准。
- **可实跑验证**：环境有 `g++ 9.4.0`（`-std=c++17 -Wall -Wextra -Wpedantic -pthread`）与 `clang++`。你可以编译运行 `./run_public_checks.sh` 或参评者新增的测试文件来验证其声称。但记住：公开检查只覆盖正常路径，**通过公开检查 ≠ 达标**。
- **识别伪修复**：重点检查是否存在"看起来修了、实际在并发/重入/边界下仍错"的情况，例如：在 mutex 内复制/移动/销毁用户 `std::function`、用 `now > last` 判 TTL、就地 mutate 共享 vector、返回内部可变对象引用、把 loader 包在全局锁内、用忙等/sleep 做同步等。

## rubric（每题 100 分）

| 维度 | 分值 | 标准 |
|------|------|------|
| 正确性 | 45 | 是否满足题面要求，核心行为与边界条件是否正确（含组合场景） |
| C++ 质量 | 20 | 是否合理使用 C++17、RAII、const、类型边界、错误处理、无未定义行为 |
| 约束遵循 | 15 | 是否遵守目录隔离、题面限制、提交格式、不改变公共 API、禁止事项 |
| 可维护性 | 10 | 是否简洁清晰、职责明确、无过度封装或无关重写 |
| 验证证据 | 10 | 是否提供可信测试、命令输出、推理或人工验证说明 |

### 硬性扣分

| 情况 | 处理 |
|------|------|
| 未提交核心代码 | 该题总分不超过 30 |
| 编造测试、命令或不存在的文件 | 该题总分不超过 40 |
| 读取或引用其他参评对象结果 | 该题总分不超过 50 |
| 违反题面明确禁止事项 | 该题总分不超过 60 |
| 大量无关重写导致无法审查 | 可维护性为 0，正确性按可验证部分给分 |
| 破坏题目结构导致无法判断 | 该题总分不超过 50 |

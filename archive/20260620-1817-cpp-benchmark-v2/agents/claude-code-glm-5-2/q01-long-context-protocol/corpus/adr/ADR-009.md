# ADR-009：协议文档取舍优先级

## 决策

当同一字段的值在多份文档中不一致时，按以下优先级取舍（高 → 低）：

**勘误表 > changelog > v6.proto 注释 > 字段词典(v4) > 历史代码**

即：勘误表最高，历史代码最低。任何与勘误表冲突的 proto 注释、changelog 或字段词典，一律以勘误表为准。

## 理由

勘误表为变更后的最终事实；changelog 记录演进；proto 注释可能滞后；字段词典停留在 v4、且部分条目已标废弃；历史实现含已知缺陷，不可作为真值来源。

## 适用范围

本优先级适用于全部 48 个实现绑定结构的全部适配字段。实现 v6 校验器时必须严格按此取舍。

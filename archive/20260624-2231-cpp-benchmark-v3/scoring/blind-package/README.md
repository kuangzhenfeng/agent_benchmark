# 匿名盲评包说明（给评分员）

你是 C++ benchmark 的盲评评分员。本目录是 `blind-package/`，含 6 份匿名提交 `P01`–`P06`。

## 评测任务

benchmark-v3（产物型架构图绘制）。每个 `Pxx/q01-batch-engine-architecture/` 含：
- `QUESTION.md`：题面与标注约定、硬性封顶条件；
- `include/` + `src/`：可编译运行的 C++17 批处理引擎 `TaskForge` 公共源码（核对架构事实的依据）；
- `diagrams/*.svg`：该匿名提交产出的 5 张 SVG（5 种风格）；
- `ANSWER.md`：该提交的架构推理与验证说明；
- 可能有 `gen_svgs.py` / `generate_svgs.py`：生成 SVG 的脚本（产物来源证据）；
- `svg_check.py`：公共 SVG 合法性检查脚本。

**你不知道、也不需要知道 P01–P06 分别对应哪个 Agent 或模型。不要尝试推断身份。**

## 可读取范围

- 本 `blind-package/` 全部内容（P01–P06 的题面、源码、SVG、ANSWER）；
- `scorer-reference/benchmark-v3.md`（私有评分参考：5 视角验收矩阵 + 5 类陷阱审查点）；
- `scorer-reference/q01_truth.json`（私有 ground truth：5 视角节点/边/分层/陷阱清单）；
- `scoring/machine-grade.json`（主 agent 已用 `grade_svg.py` 跑出的客观覆盖率，见下）。

**禁止读取**：`mapping.private.md`、`redaction-log.md`、`agents/`、`submissions/`、`participants.md`、`questions/` 以及本轮 `benchmark/` 根目录的其他文件。

## 评分维度（每题 100 分）

| 维度 | 分值 | 说明 |
|------|------|------|
| 正确性 | 45 | 架构事实是否准确。**主轴**：直接采用 `machine-grade.json` 的 `coverage_score_out_of_45`（5 视角节点/边覆盖率加权 + 陷阱加分）。可在 ±3 内调整：如发现机器漏判的明显架构错误、或标注归一化未覆盖但你核对源码确认正确的命名。引用源码事实与机器分共同说明。 |
| SVG 产物质量 | 20 | 与正确性相对独立的"图本身质量"：SVG 是否可渲染、标注是否规范（data-style/data-role/data-id/data-from-to-kind）、5 视角是否齐全且不混淆、视觉表达是否清晰（不是"好不好看"，而是结构是否可读）。主观，不评价配色。 |
| 约束遵循 | 15 | 是否遵守：只改 diagrams/ANSWER（不碰引擎源码、QUESTION、svg_check）、5 风格齐全且文件名/data-style 符约定、节点≥6/边≥4、不联网无外部依赖。 |
| 可维护性 | 10 | ANSWER 是否简洁清晰、推理职责明确、无过度封装或无关重写；生成脚本（若有）是否可复现。 |
| 验证证据 | 10 | 是否提供 `run_public_checks.sh` / `svg_check.py` 的实际输出、推理是否引用源码位置、是否诚实说明未覆盖项。 |

## 硬性扣分（题面"硬性封顶条件"）

| 情况 | 上限 |
|------|------|
| 5 种风格缺任一种，或文件名/data-style 不符约定 | 该题 ≤ 60 |
| 任一 SVG 非合法 XML / 根非 `<svg>` / 无可渲染尺寸 | 该题 ≤ 60 |
| 把 C++ 源码/注释原样当图（foreignObject 贴代码、纯文字堆砌） | 该题 ≤ 50 |
| 5 张全无 data-* 标注 | 该题 ≤ 40 |
| 修改引擎源码/QUESTION/svg_check.py | 该题 ≤ 60 |
| 编造测试、命令或不存在的文件 | 该题 ≤ 40 |
| 读取或引用其他参评对象结果 | 该题 ≤ 50 |

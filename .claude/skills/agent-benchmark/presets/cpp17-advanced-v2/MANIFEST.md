# cpp17-advanced-v2

这是面向长业务协议、闭世界实现和协议适配的 C++17 基准预设。协议本体使用 C++ 结构体、JSON 与 Protobuf 表达；标记文档只用于题目导航、作答说明和评测流程。

| 题目 | 类型 | 推荐时间盒 | 协议工件 | 协议长度 | 评测重点 |
|------|------|------------|----------|----------|----------|
| q01-settlement-journal | 组合缺陷修复 | 65 分钟 | 业务协议/结算账本协议.hpp | 1,000 行 | 结算幂等作用域、版本优先级、金额边界、文本所有权、审计重入 |
| q02-benefit-allocation | 实现 | 90 分钟 | 业务协议/权益分配业务协议.json | 3,000 行 | 单条与批次回放、临时账本、原子提交、额度优先级、回调隔离 |
| q03-protocol-adaptation | 协议适配 | 210 分钟 | 协议一/结算适配源协议.proto 与 协议二/结算适配目标协议.json | 各 5,000 行 | 多结构逐字段迁移、近似名称辨别、范围控制、反机械复制 |

## 公开性与边界

- 所有业务规则都在题面或上述协议工件中公开；评分不得使用未公开的私人条件。
- 协议长度用于模拟业务合同的检索与交叉核对成本，不是用无意义文本替代工程难度。
- Q3 的协议一变更映射只说明迁移动机和卡片定位；协议二的实现绑定卡片才是目标字段真值。
- 本预设不保证任何模型或人类必然失败或通过。能力结论只能来自同题、隔离作答与匿名盲评。

## 使用规则

- 用户明确选择 cpp17-advanced-v2 时，完整复制三个题目目录到本轮 questions，再复制到每个 agents 目录。
- 业务协议工件、QUESTION.md、ANSWER.md、起始代码与 tests 目录都是公开题源；所有参评对象必须拿到字节等价副本。
- 生成与作答阶段绝不能复制私有评分参考到 questions、agents 或 submissions。
- 全部参评对象停止作答后，才把对应私有评分参考复制到本轮 scorer-reference。
